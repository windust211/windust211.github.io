git常用操作

git 是什么？
  • 源代码管理器 Linux
    初始化仓储；
  • git init 初始化；

配置 用户名 密码；

  • 备份在 git文件里；
    • git config –global user.name “”;
    • git config –global user.email “”
代码放到仓库里；
//工作区
git add //放到大门口(暂存区)
git commit //放到房间里； 全部放；// 版本库
//说明
git commit -m '代码更新了'；

vim 里面如何退出

esc->:->q
esc->:->q!

状态查看

git status

添加所有文件

git ./

直接加到版本库

git commit --all -m 'all'

设置忽略文件

添加 .gitignore文件
//添加过程；(直接写不出来。。)
touch .gitignore

查看日志

git log
git log 
--oneline  //一行显示一条

版本回退；

  • git reset –hard Head~0 //往前回退一次；
    // 0表示索引；

  • 通过版本号回退

    git reset --hard f48bad4 //版本号
git reflog //看看所有操作；

创建分支，合并，查看，合并；

  • 创建分支；
    • git branch dev1
  • 查看分支,绿色是当前位置
    • git branch
  • 切换分支；
    • git checkout dev1
    • git checkout master
  • 合并分支
    • git merge dev1 (切换到master 合并 dev1)

处理冲突 （两个都修改,在合并的时候出现这个问题）

- 删除分支；git branch -d dev1 (切换到别的分支)
-找到冲突的部分，删除，重新commit然后合并；

推送远端服务器

- git push origin master

获取github上面的代码

-git clone ''

拉取远程服务器代码；

  • clone 上面都有都会取下来
  • pull 只拉代码，不拉 .git
  • 第一次使用 clone；
  • 后面都用pull;
    • 使用 clone，全部的代码包括里面的.git文件;
    • 如果使用 pull,是修改部分的代码；

https 方式提交 vs ssh方式提交；

  • use https/ use ssh;
  • ssh
    • 生成文件 公钥 ，私钥 有关联
    • 私钥给自己，公钥给github(在github里面配置)
    • 生成公钥，私钥
      1. ssh-keygen -t rsa -C "js.rookie211@gmail.com"
      2. 默认位置，直接enter
      3. 输入密码； 可以不设置；

      6. 然后ok。。。

多账户操作获取代码时候，出现覆盖，冲突；

  • 工作区在pull的时候，会被覆盖；
  • 在commit里面提交的时候，出现conflict；
  • 提交代码的时候，先pull后 push;
